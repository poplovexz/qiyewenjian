# 🚀 快速启动指南

## 一键启动所有服务

### 最简单的方式

```bash
cd /var/www
./start_all.sh
```

就这么简单！脚本会自动：
- ✅ 启动 PostgreSQL 数据库
- ✅ 启动 Redis 缓存
- ✅ 启动后端 API（端口 8000）
- ✅ 启动前端应用（端口 5174）

### 访问应用

启动完成后，在浏览器中打开：

**前端应用**: http://localhost:5174

**后端 API 文档**: http://localhost:8000/docs

## 停止所有服务

```bash
./stop_all.sh
```

## 查看服务状态

```bash
./status_backend.sh
```

## 查看日志

```bash
# 后端日志
tail -f /tmp/backend_8000.log

# 前端日志
tail -f /tmp/frontend_5174.log
```

## 常见问题

### 1. 首次使用需要配置

如果是第一次使用，需要先配置环境变量：

```bash
cd /var/www/packages/backend
cp .env.example .env
vim .env  # 编辑配置文件
```

### 2. 端口被占用

如果提示端口被占用，先停止旧服务：

```bash
./stop_all.sh
./start_all.sh
```

### 3. 服务启动失败

查看日志找出原因：

```bash
# 后端日志
tail -f /tmp/backend_8000.log

# 前端日志
tail -f /tmp/frontend_5174.log
```

## 只启动后端

如果只需要启动后端服务：

```bash
./start_backend_all.sh
```

## 脚本说明

| 脚本 | 功能 |
|------|------|
| `start_all.sh` | 启动前端 + 后端所有服务 |
| `stop_all.sh` | 停止所有服务 |
| `start_backend_all.sh` | 只启动后端服务 |
| `stop_backend_all.sh` | 只停止后端服务 |
| `status_backend.sh` | 查看服务状态 |

## 服务端口

| 服务 | 端口 |
|------|------|
| 前端应用 | 5174 |
| 后端 API | 8000 |
| PostgreSQL | 5432 |
| Redis | 6379 |

## 开发流程

### 每天开始工作

```bash
cd /var/www
./start_all.sh
```

### 修改代码

- 后端代码修改后会自动重载（热重载）
- 前端代码修改后会自动刷新

### 结束工作（可选）

```bash
./stop_all.sh
```
