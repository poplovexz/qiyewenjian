权限页面响应式数据测试

问题描述

权限列表页面存在以下问题：

  • 首次打开页面看不见内容
  • 点击其他菜单后再点击就出现了
  • 刷新页面又不见了
  • 没有报错

问题原因分析

根本原因：Vue 3响应式数据解构问题

原代码问题： // ❌ 错误的写法 - 直接解构会丢失响应式 const { permissions, loading, total, currentPage, pageSize } = permissionStore 修复后： // ✅ 正确的写法 - 使用storeToRefs保持响应式 import { storeToRefs } from 'pinia' const { permissions, loading, total, currentPage, pageSize } = storeToRefs(permissionStore)

问题详细解释

1. 响应式丢失： - 直接解构Pinia store会导致响应式丢失 - 解构后的变量变成普通值，不再响应数据变化 2. 首次加载问题： - 组件挂载时，store中数据可能是初始值（空数组） - 解构后的permissions固定为空数组 - 即使API调用成功更新了store，组件也不会重新渲染 3. 切换菜单后显示： - 切换到其他页面再回来时，组件重新创建 - 如果store中已有数据，此时解构到的是有数据的数组 4. 刷新后消失： - 刷新页面重置store状态 - 解构到的又是空数组
